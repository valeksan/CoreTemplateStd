# CoreTemplateStd v0.5.2

This patch release fixes the non-cooperative task behavior in the Qt GUI example and removes duplicate timeout reporting.

## Highlights

- New CMake option: `CORETEMPLATE_ENABLE_UNSAFE_FORCE_TERMINATION`.
- Safe std-only builds still use cooperative cancellation plus timeout reporting.
- Builds with unsafe force termination can use native thread termination APIs where available.
- `ExampleQtGuiApp` enables unsafe force termination by default, so the non-cooperative demo task can be interrupted after its stop timeout.
- Repeated cancel/terminate requests no longer emit duplicate stop-timeout events for one task.

## Safety Note

Unsafe force termination can stop a non-cooperative task, but it may skip destructors and task cleanup code. Prefer cooperative cancellation for production code.
