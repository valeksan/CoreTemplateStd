# CoreTemplateStd v0.3.0

This release restores an optional Qt Widgets GUI example on top of the Qt adapter.

## Highlights

- New optional CMake option: `CORETEMPLATE_BUILD_QT_GUI_EXAMPLE`.
- New target: `ExampleQtGuiApp`.
- GUI example uses `CoreQtAdapter` rather than adding Qt back into `core.h`.
- CI builds the Qt adapter and GUI example in the Qt job.

## Build The GUI Example

```cmake
cmake -S . -B build/qt_gui \
  -DCORETEMPLATE_BUILD_EXAMPLE=OFF \
  -DCORETEMPLATE_BUILD_QT_GUI_EXAMPLE=ON
cmake --build build/qt_gui --target ExampleQtGuiApp
```

## Core Boundary

The std-only target remains unchanged:

- `core.h` has no Qt includes.
- `CoreTemplateStd::CoreTemplateStd` does not require Qt.
- Qt is required only for `CoreTemplateStd::QtAdapter` and `ExampleQtGuiApp`.
