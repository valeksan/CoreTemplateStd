# CoreTemplateStd v0.2.0

This release adds an optional Qt adapter while keeping the core target std-only.

## Highlights

- New optional CMake option: `CORETEMPLATE_BUILD_QT_ADAPTER`.
- New target: `CoreTemplateStd::QtAdapter`.
- New `CoreQtAdapter` QObject bridge.
- Adapter re-emits std-only core callbacks as Qt signals.
- Adapter smoke test and Ubuntu CI job.

## Core Boundary

`core.h` still has no Qt includes and the `CoreTemplateStd::CoreTemplateStd` target does not require Qt.

Qt is required only when building the optional adapter:

```cmake
set(CORETEMPLATE_BUILD_QT_ADAPTER ON)
add_subdirectory(CoreTemplateStd)

target_link_libraries(your_qt_target PRIVATE
    CoreTemplateStd::QtAdapter
)
```

## Signals

`CoreQtAdapter` emits:

- `startedTask(TaskId, TaskType, QVariantList)`
- `finishedTask(TaskId, TaskType, QVariantList, QVariant)`
- `terminatedTask(TaskId, TaskType, QVariantList)`
- `stopRequestedTask(TaskId, TaskType, QVariantList)`
- `stopTimedOutTask(TaskId, TaskType, QVariantList, TaskStopTimeout)`
