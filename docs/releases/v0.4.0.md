# CoreTemplateStd v0.4.0

This release makes event-loop integrations wake-driven instead of polling-driven.

## Highlights

- New std-only API: `Core::setWakeCallback` / `Core::clearWakeCallback`.
- `CoreQtAdapter` uses queued Qt delivery for immediate wakeups.
- `CoreQtAdapter` uses `QTimer::singleShot` for delayed core events.
- `ExampleQtGuiApp` no longer needs a periodic polling timer.

## Core Boundary

The wake callback is a standard C++ function hook:

```cpp
core.setWakeCallback([](TaskStopTimeout delayMs) {
    // Wake your owner event loop now or after delayMs.
});
```

No Qt headers are added to `core.h`.

## Why

The core still owns the event queue and delayed owner events, but integrations can now wake their event loop only when there is work to deliver. This avoids a constant GUI polling timer while keeping `Core::processEvents()` explicit and std-only.
