# CoreTemplateStd v0.1.0

Initial std-only release of CoreTemplateStd.

## Highlights

- Header-only C++17 core with no required Qt dependency.
- CMake package target: `CoreTemplateStd::CoreTemplateStd`.
- Compatibility CMake target: `CoreTemplate::CoreTemplate`.
- Task callbacks replace Qt signals.
- Task payloads use `std::any` and `std::vector<std::any>`.
- Cooperative cancellation, stop timeout events, grouping, and queued task execution are covered by std-only tests.
- CI covers Linux, macOS, Windows, installed-package smoke tests, and Linux ASan/UBSan.

## Recommended CMake Usage

```cmake
include(FetchContent)

FetchContent_Declare(
    CoreTemplateStd
    GIT_REPOSITORY https://github.com/valeksan/CoreTemplateStd.git
    GIT_TAG v0.1.0
)

FetchContent_MakeAvailable(CoreTemplateStd)

target_link_libraries(your_target PRIVATE CoreTemplateStd::CoreTemplateStd)
```

Installed package consumers can include:

```cpp
#include <CoreTemplateStd/core.h>
```

## Migration Notes

- Replace Qt signals such as `finishedTask` with callback setters such as `onFinished`.
- Replace `QVariant`/`QVariantList` payload reads with `std::any_cast<T>` from `TaskResult` and `TaskArgs`.
- Call `Core::processEvents()` from the managing thread to deliver queued callbacks.
- Construct `Core` directly; `QObject` parent ownership is no longer part of the core API.
